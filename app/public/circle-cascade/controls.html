Parametric shape controls (inert in empirical modes — driven by measurements)
Falloff (k)1–6 ●Controls how quickly the circles thin out as you move from top to bottom. Low values (around 1) spread circles fairly evenly down the canvas; high values (around 6) pile them heavily at the top and leave the bottom sparse. Each unit of k cuts the local density by a constant factor per row. ○Not used. The top-to-bottom density pattern is copied row-by-row from the reference image. ○Not used. Density pattern comes from the reference image. ○Not used. Density pattern comes from the reference, plus the optimal-transport refinement nudges every circle toward a reference target.
Size top (px)6–60 ●The average circle radius at the very top of the canvas. Think of it as "how big are the biggest circles up top." Circles are then scaled smoothly down toward the bottom value as they go lower. ○Not used. Circle sizes come from row-by-row measurements of the reference image. ○Not used. Sizes come from reference measurements. ○Not used. Sizes come from reference measurements.
Size bottom (px)1–24 ●The average circle radius at the very bottom of the canvas. Size shrinks linearly between Size top and Size bottom as you move down, so a circle in the middle of the canvas will be roughly the average of the two. ○Not used. ○Not used. ○Not used.
Size jitter0–0.8 ●How much each individual circle's size is allowed to wander away from the row-average. 0 means every circle on a row is exactly the same size; 0.8 gives you a wide mix of noticeably smaller and bigger circles. Adds visual variety without changing the overall gradient. ○Not used. Size variation is baked into the reference statistics. ○Not used. ○Not used.
Gap (px)0–20 ●The minimum empty space forced between the edges of any two neighboring circles. Only takes effect when allow overlap is off. Raise it to give the layout more breathing room; push it too high and some circles may fail to find a spot and won't be placed. ○Not used. Minimum spacing is taken directly from the reference — whatever distance it avoids, this algorithm avoids too. ○Not used. Spacing comes from matching the reference's pair-correlation curve. ○Not used.
Cluster bias0–0.8 ●The probability that a new circle is placed near an existing one rather than sampled freely from the density field. Higher values produce clumpier, grape-like groupings; lower values produce a more even scatter. Great for shifting the feel between "tight bunches" and "loose sprinkle" without changing the density gradient. ○Not used. ○Not used. Clumpiness is inherited from the reference's measured pair-correlation function. ○Not used.
Horizontal noise0–1 ●Adds slow, wavy variation in density across the horizontal direction, so rows aren't uniformly packed left-to-right. At 0 every row is flat and even; at 1 you get strong vertical streaks of denser and sparser circles. Helps avoid an obviously algorithmic striped look. ○Not used. ○Not used. ○Not used.
Empirical modes replace the shape sliders with statistics measured from the reference image: λ(y) density, R(y) radius-by-row, g(r) pair correlation, and (for OT) the full reference point cloud nudged via Paulin 2020 sliced optimal transport.
